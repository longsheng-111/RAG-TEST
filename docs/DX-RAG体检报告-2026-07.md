# DX-RAG 项目体检报告与后续发展顾问

> 体检日期：2026-07-17 · 代码基线：当前工作区（frontend Next.js 14.2.24 + antd 5.22.7 / backend FastAPI）
> 评判基准说明：仓库中未找到独立的《风格基准文档》文件（已检索 `docs/`、根目录全部 md）。本报告以 `frontend/app/globals.css:9-52` 的 token 层与工具类层作为**事实风格基准**（暖纸作业本 × 硬边框 × 复古 RPG），这与提示词描述的规范一致。若另有书面基准文档，请补充后复核第 1 维度。

---

## TL;DR

**最痛的 3 个问题**
1. **主题三轨并存、同名类五处定义**——ConfigProvider token、`globals.css` 的 `!important` 覆盖、组件内 `<style>` 再覆盖同时生效，`.op-btn`/`.op-card` 等类在 5 个文件里被重复定义且已发生肉眼可见的漂移（如 `Sidebar.tsx:27` 的 `SUNKEN='#EDE1C8'` 与全局 `#F0E3C6` 不一致）。任何改色都要改 N 处，是风格维护的最大负债。
2. **键盘与可访问性近乎裸奔**——全部可点击 div（会话卡片、引用角标、来源卡片、聚类头）无 `tabIndex`/键盘事件，会话卡操作按钮仅 hover 可见，`--ink-faint` 正文对比度约 2.6:1（WCAG AA 要求 4.5:1），Mac 上 Cmd+Enter 不生效。
3. **错误处理静默化**——5 处 `catch { /* ignore */ }` 吞错（QAPanel 3 处、FileManager/FileUpload 各 1 处），3 处浏览器原生 `alert()`，失败只闪 toast 无持久错误态，用户遇到故障无任何可循线索。

**最值的 3 个机会**
1. **引用纠错数据是金矿**——`POST /api/feedback` + `services/feedback.py` 已把用户的"替换引用/标记不准确"落盘并注入后续回答，但无读取端点、无管理界面。补上查询端点即可做"纠错记录/质量看板"，是零新依赖的差异化。
2. **面试报告呼之欲出**——后端 `exam_state` 已持久化总分/单题分/weak_points/summary，前端只展示一次就丢。复用 `GET /api/exam/{id}` + 已有的 Markdown 导出模式即可做出可回看的面试报告，直接命中求职者核心诉求。
3. **`/api/stats` 与会话元数据闲置**——统计端点、token 用量、消息数都已存在但前端未消费，拼一个"学习进度/成就"页成本极低，且与 RPG 世界观天然契合。

## 第六章 产品后续发展规划

### 近期（2–4 周）· 体验闭环

**1. 引用纠错记录与质量回收**
- 用户价值：我标记过"不准确"的引用、做过的替换，集中可见可追溯——知道系统有没有真的听我的。
- 复用：`services/feedback.py`（JSON 落盘已存 type/原 chunk/目标 chunk/note/时间）、`POST /api/feedback`（`routes.py:512`）、`QAPanel.tsx:957-1002` 的提交链路。缺的只是一个 `GET /api/feedback`（按 session 或全局查询）+ 一个管理页（可复用 `FileManager` 的表格模式）。

- 用户价值：每题答完立刻知道"命中了哪几个得分点、差在哪"，而不是一个干巴巴的分数。

- 风险：评分由 LLM 一次性同步生成（`routes.py:632-678`），等待 10s+，需先解决等待体验（维度 5.1 的阶段提示）。

- 用户价值：Cmd+Enter 能发送、后端挂了能看到错误、空态不再"报错"。

**2. 基于资料的错题本 / 抽认卡**
- 用户价值：面试中"未命中"的得分点、QA 中标记过"不准确"的知识点，自动生成抽认卡，考前刷一遍。RPG 世界观里这就是"怪物图鉴/错题讨伐"。
